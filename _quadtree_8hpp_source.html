15/*
16 To parallelize the quadtree, we need to decompose the layers in subcells that we are sure don't interact with each others
17 * for the lowest levels (leafs) we know only the inside objects can interact so we save for each node the number of pairs
18 * for the uppers levels, we have to care that the pairs with sub-levels don't interact n the same object, we can keep the
19 same logic by decomposing the pairs by levels with the depths and recording the number of pairs in this current node...
20 * then we only have to climb these layers while parallelizing the nodes...
21*/
